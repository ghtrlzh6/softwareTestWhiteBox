# 成员 D 交付物：AI 与传统白盒工具对比及总结报告

## 1. 本成员职责
- **对比分析**：系统性对比 LLM 辅助测试与传统静态/动态分析工具的优劣。
- **局限归纳**：总结实验过程中发现的 LLM 生成局限性及“幻觉”成因。
- **方案展望**：针对现有工具链的痛点，设计并提出“自愈式测试闭环”等改进方案。

## 2. AI 与传统白盒测试工具的对比分析

通过对 `Got` 库调度逻辑的实战验证，我将 LLM 生成测试与传统工具（如 Istanbul、SonarQube 或符号执行工具）的对比总结如下：

| 维度 | 传统白盒工具 (Traditional Tools) | AI 辅助测试 (LLM-Based) |
| :--- | :--- | :--- |
| **测试输入生成** | **弱**。通常只能监测路径，无法自动构造满足复杂分支条件的业务数据。 | **强**。能理解变量语义，自动构造符合业务逻辑的参数、Mock 对象及异步状态。 |
| **逻辑理解深度** | **零**。基于语法树的硬性扫描，无法理解“回退（Fallback）”或“重试”的业务意图。 | **高**。能通过自然语言理解函数的业务背景，生成具有针对性的功能验证用例。 |
| **严谨性与确定性** | **100%**。基于数学模型和物理执行轨迹，结果可复现，无逻辑幻觉。 | **波动性**。受模型随机性影响，同一 Prompt 可能产生质量不一的代码。 |
| **工程接入成本** | **高**。需要复杂的手动环境配置、依赖注入编写以及反射权限处理。 | **低**。通过 Prompt 即可快速生成 Mock 模板，显著降低了白盒测试的起步门槛。 |

---

## 3. 实验发现：LLM 的三大深层局限性

尽管本项目在 `getRequestFunction` 上达成了极高的物理覆盖率，但在自动化流水线运作中，我识别出了以下瓶颈：

1. **上下文碎片化（Context Fragmentation）**：
   当测试逻辑涉及跨文件的深层类继承或 Node.js 原生 C++ 模块（如 `http2` 底层实现）时，AI 往往因为无法获取全局代码图谱而产生“逻辑断层”，导致生成的 Mock 代码与实际环境脱节。
2. **“幻觉覆盖”与虚假通过（False Positives）**：
   AI 倾向于编写“自圆其说”的测试。它有时会构建一个过于理想化的“无菌室”环境，使得测试在 Mock 层面完美运行，但实际上由于 Mock 逻辑太简单，根本没有触达源码中真实的边界条件。
3. **Token 限制下的“逻辑裁剪”**：
   面对长函数，LLM 为了符合输出长度限制，会策略性地忽略一些边缘分支（Corner Cases）。例如，它常忽略 `is.promise(result)` 逻辑中关于 `Reject` 异常的复杂处理。

## 4. 未来改进方案：自愈式测试闭环（Self-Healing Loop）

针对上述痛点，我提出以下三点改进方案，作为下一阶段工具链迭代的路线图：

### A. 引入“生成-校验-反馈”闭环架构
当前的流水线是单向的。理想模型应包含一个**反馈回路**：
- **触发**：当物理覆盖率工具（V8）报告某行未被覆盖或 Vitest 运行报错。
- **反馈**：脚本自动提取报错日志及未覆盖行的上下文。
- **修复**：将失败信息作为“负面反馈”重新喂给 LLM，强制其针对特定逻辑空洞重新生成测试代码。

### B. 结合静态分析（AST）增强 Prompt 精度
在调用 LLM 之前，先利用抽象语法树工具提取函数的**控制流图（CFG）**和依赖表。将这些结构化数据而非纯文本代码输入给 AI，能大幅降低其在生成参数时的随机性，提升路径命中的精准度。

### C. 混合测试策略：AI 驱动生成 + 符号执行驱动约束
利用**符号执行（Symbolic Execution）**解算复杂的路径约束，获得满足分支条件的精确数学解；再由 **LLM** 将这些枯燥的数学约束“翻译”成具备业务逻辑、可读性强的测试脚本，实现“机器严谨性”与“AI 语义表达”的互补。

---

## 5. 本阶段实验总结
- **结论一**：LLM 是极其优秀的**测试用例生成器（Generator）**，能极大降低 Mock 编写成本，但它不能作为**覆盖率事实来源（Truth Source）**。
- **结论二**：白盒测试自动化的终极形态应是“AI 推理驱动 + 物理引擎审计”的混合模式。
- **结论三**：通过本项目设计的流水线，我们成功将原本需要数小时的手工人工作业，转化为分钟级的自动化生成与验证过程。
